//สร้างกลุ่ม
สร้างโดยนักศึกษา
	1. สร้างแล้วเพิ่มสมาชิกครบ 
		- สถานะเป็น 2 (Wait Advisor)
	2. สร้างแล้วสมาชิกไม่ครบ
		- สถานะเป็น 1 (Draft)
สร้างโดยอาจารย์
	1. สร้างแล้วเพิ่มสมาชิกครบ 
		- สถานะเป็น 3 (Inprogress)
	2. สร้างแล้วสมาชิกไม่ครบ
		- สถานะเป็น 1 (Draft)

//เข้าร่วมกลุ่ม
1. ถ้ามีกลุ่มแล้วจะเข้าร่วมกลุ่มอื่นไม่ได้
2. ถ้าสมาชิกกลุ่มนั้นเต็มแล้วจะเข้าร่วมไม่ได้
3. ถ้า Project นั้นๆ อยู่ในสถานะ 3 (Inprogress) แล้วจะเข้าร่วมกลุ่มไม่ได้

//ออกจากกลุ่ม
1.ออกจากกลุ่มขณะที่สถานะเป็น Wait Advisor เปลียนสถานะโปรเจ็คเป็น Draft

//อนุมัติเป็นที่ปรึกษา
1. สามารถอนุมัติได้แค่โปรเจ็คที่เป็นสถานะ 2(Wait Advisor)
2. หน้า "โปรเจ็คทั้งหมด" จะเห็นรายการที่เป็นทั้ง (1)Draft และ 2(Wait Advisor) //อาจจะให้เห็นแค่ Wait Advisor เพราะจะมีปัญหาตอน Approve แบบ หลายกลุ่ม

//ส่งเอกสาร
1. เริ่มส่งเอกสารได้เมื่อ Project Status เป็น (3)Inprogress
2. ส่งได้ก็ต่อเมื่อผ่านเงื่อนไข Prerequisite ตามที่อาจารย์ประจำวิชากำหนด
3. แต่ละ CE สามารถส่งกี่ครั้งก็ได้โดยแต่ละครั้งจะแยกกัน
4. ส่ง CE ใน Format PDF เท่านั้น
5. Comment เอา Comment ของทุกครั้งที่ส่งมาแสดงเรียงจากใหม่ไปเก่า
6. ส่งเอกสาร สถานะเริ่มต้นเป็น -> Wait Advisor, ที่ปรึกษาอนุมัติ -> Wait Instructor, ประจำวิชาอนุมัติ -> Approved 
7. ถ้าถูก Reject ต้องส่งใหม่ โดยทำ Flow ข้อ 6 ใหม่
8. แต่ละ CE มีกำหนดส่ง โดยอาจารย์ประจำวิชาเป็นคนกำหนด
9. เมื่อเกินกำหนดส่งแล้วจะส่งไม่ได้ แต่อาจารย์ประจำวิชาสามารถขยายเวลาส่งได้

//ขอสอบ
1. ถึงช่วงเวลาที่อาจารย์ประจำวิชากำหนด
2. ผ่าน CE ตามที่อาจารย์ประจำวิชากำหนด
3. เมื่อส่งฟอร์มขอสอบ อาจารย์ที่ปรึกษาและประจำวิชาต้องอนุมัติ
4. อาจารย์ท่านอื่นๆ มาเลือกเป็นกรรมการสอบ กลุ่มที่อาจารย์ที่ปรึกษาและประจำวิชาต้องอนุมัติแล้ว
